Project information

Project detail

I did the project within a six-week schedule. This project is a management system for project operation expenses in the intership company. There were 15 members from different computer universities in this project. There were two team leaders in this project. I was one of the team leaders. This project was the best challenge for me. We used Spring Boot to develop this project.

There are five roles (project operation member, project manager, department head, budget manager and admin) in Project Operation Expense Management System. Project operation members can request expenses for their projects. Project manager, department head and budget manager can make decisions whether accept or reject on the requests from the members. Admin role can do all permissions.

Our project is actually needed in the company. The web application will help the management processes of the company. Before our development, the company managed in some process manually with paper, email, excel file and so on. With our web application, the company can reduce workload, time and costs.

We met our customer, budget manager to get requirements for the project. She wanted to replace their manual process of management with software. Whenever we met with our customer, we wrote meeting minutes as a record.

After collecting requirements for the project, we thought about how to replace manual processes with software. We sketched UI designs for projects according to customer requirements. After sketching UI design, the next state is UI prototyping. We used Adobe XD software to draw UI prototypes.

After the prototyping stage, we made detailed designs for our project. There are detailed specifications for every item and every function in every page. We had to implement the project correctly as written in detail design. Whenever one stage of the development process was finished, we did unit testing to clear the bugs or errors in our project. After integrating the whole system, we also did integration testing and user acceptance testing.

Finally, I had to know how to develop an application in real life. And, I well understood how to develop the whole software development. I got not only the project experiences but also the management experiences from this project.
